Castle Escape

Proposal

High Concept

This game will be like "The World's Hardest Game" but it will have more of a story. Instead of just having different colored shapes, everything on screen will actually be something. For example, the player will move a character across the screen rather than a square and the user will pick up things like keys to open doors instead of just colored circles. The player will move their character through each level collecting keys and also avoiding ghosts on their journey to escape the castle.

Genre

This will be a strategy game that makes the player plan paths and time their movements. It will be an avoidance game because the user will be attempting to avoid ghosts while they move throughout the castle. Lastly, there will be escape and adventure elements to the game as the player moves throught the castle looking to escape.

Platform

This game will be designed and developed for desktop-only playing.

Story

The theme of the game is for the player to move their character throught the levels trying to run away from the ghosts and escape from the castle. The mood of the game will be excitement as the player will find joy in completeing levels and reaching the exit of the castle. The player will progress through each level of the game by finding a key in a room and using it to open a door. The user will be tempted to continue playing as they will be driven to escape the castle and reach the end.

Aesthetics

The graphics style will be cartoonish and I may use Adobe Illustrator to create simple vector art for some elements. The sound will be very simple with audio playing for finishing a level, collecting a coin, and getting hit by a ghost. I may also add simple background music.

Gameplay

The player will be able to move the character using WASD and that will be the main functionality of the game. I may add some sort of power-up that the player can collect and then use the space-bar to use. The player will learn the movement options on the screen of the first level as their will be some simple text to explain it. If I choose to incorporate a power-up on some levels I will have text to explain it on the level it's introduced on.

Documentation

My Process

When I started working on this project I first took some time to come up with an idea for a game that I thought I was capable of building and that I would actually find fun to play. I decided to somewhat recreate "The World's Hardest Game" but to give it more of a story. When I started coding I began by creating the different screens that would be shown at different times depending on where the player was. After coding the buttons to move from screen to screen I then began creating the actual functionality of the game. This started by creating a character on each screen that could be moved using the mouse. I then coded the zombies that the player would need to avoid hitting. Then I created the box that the player can touch to complete each level and lastly I created the keys that the player need to collect to make the box appear. Up until this point I was just using images I found online as the main character, zombies, and keys so once I knew the functionality was mostly working I went into Adobe Illustrator and created all the graphic elements that the game uses. At this point I had only actually built out the first level so I then began to code other levels and in doing so I created two new classes for zombies that allow them to move in different ways. The end of my process included messing with the smaller details such as the typefaces, exact positioning of textand buttons, and the color palette. One of the final things I did was search for sound effects that I could use and then implement them into my code.
